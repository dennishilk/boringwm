# Contributing to BoringWM

Thanks for your interest in BoringWM.

BoringWM is intentionally minimal and conservative.
Not every feature request will be accepted.

## Bug Reports
- Use GitHub Issues
- Include:
  - distribution
  - X11 environment
  - steps to reproduce
  - logs if available

## Feature Requests
- Keep them small and focused
- Avoid feature creep
- Explain the problem, not just the solution

## Pull Requests
- Small, focused changes only
- No large refactors without prior discussion
- Code must build on Debian Stable
